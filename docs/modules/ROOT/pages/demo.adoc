= Demo Setup
:page-title: Demo Setup

The repository includes a full demo setup under `hack/demo/`.

The demo now deploys the operator via Helm chart (`charts/resource-action-operator`).

== Requirements

- kind
- kubectl
- docker
- helm
- openssl

== Full Run

[source,bash]
----
hack/demo/run-all.sh
----

== Step By Step

[source,bash]
----
hack/demo/setup-kind.sh
hack/demo/deploy-operator-kind.sh
hack/demo/apply-receivers.sh
hack/demo/apply-resourceactions.sh
hack/demo/trigger-actions.sh
hack/demo/show-logs.sh
----

== Operator Image For Demo

By default, the deploy script builds and loads:

- `IMG=resource-action-operator:dev`

Override image tag if needed:

[source,bash]
----
IMG=resource-action-operator:dev hack/demo/deploy-operator-kind.sh
----

The deploy script maps `IMG` to Helm values:

- `image.repository`
- `image.tag`
- `image.pullPolicy=IfNotPresent`

== Cleanup Behavior

`hack/demo/cleanup.sh` removes:

- demo workloads and ResourceActions
- Helm release in `resource-action-operator-system`
- CRD `resourceactions.ops.yusaozdemir.de`

This gives a clean base for re-runs and integration test cycles.

== Cleanup / Reset

[source,bash]
----
# soft cleanup
hack/demo/cleanup.sh

# hard cleanup (includes cluster delete)
DELETE_CLUSTER=true hack/demo/cleanup.sh

# soft reset
hack/demo/reset.sh

# hard reset
FULL_RESET=true hack/demo/reset.sh
----
