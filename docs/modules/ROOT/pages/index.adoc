= Resource Action Operator
:page-title: Overview

Resource Action Operator executes declarative HTTP and Job actions on Kubernetes events.

== Core Concepts

- `ResourceAction` selects target resources by Group/Version/Kind
- events (`Create`, `Update`, `Delete`) define when actions are triggered
- optional filters (name/namespace/labels) narrow trigger scope
- HTTP actions support retries, timeouts, TLS, and templated bodies
- Job actions create Kubernetes Jobs with user-provided images, commands, or scripts

== Deployment Modes

- Helm is the recommended install path for cluster deployment.
- Helm default mode works without cert-manager.
- Admission webhook mode is optional and requires cert-manager.
- Kustomize deployment remains available for development workflows.
- Additional Helm charts are available for generic and job-focused `ResourceAction` resources.

== Repository Highlights

- `charts/resource-action-operator` installs the operator and CRD
- `charts/resource-action` creates generic HTTP or Job-based `ResourceAction` objects
- `charts/resource-action-job` creates job-focused `ResourceAction` objects
- `examples/` contains focused manifest examples for common setups
